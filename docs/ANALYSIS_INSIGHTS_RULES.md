## User-Facing Wording Rules

- Every insight sentence should include evidence naturally: record, sample size,
  win rate, score gap, streak length, ELO expectation gap, money, or days absent.
- Do not show raw engineering wording to users:
  - Avoid `baseline`, `impact`, `performance score`, `deuce`, or prediction
    "machine" wording.
  - Use `tỷ lệ thắng dự tính`, `mức trước trận`, and `kéo qua 11 điểm`.
  - For relationship-impact copy, do not expose `impact` as `X điểm`; users can
    misunderstand it as on-court score margin. Prefer qualitative wording such
    as `kết quả thực tế cao hơn hẳn mức trước trận`, `đánh hiệu quả hơn hẳn
    khi đi chung`, or `tụt rõ so với mức thường thấy`.
  - For upset/probability copy, prefer `tỷ lệ thắng trước trận`, `cửa thắng
    trước trận`, `cửa dưới`, or `mâm dưới`; avoid mentioning ELO unless the
    insight is explicitly about ELO rank/rating.
- Sports idioms such as `máy cày` or `máy test vợt` are allowed because they do
  not refer to the prediction system.
- Strong teasing words such as `out trình`, `đóng hòm`, `át vía`, `báo thủ`, or
  `gánh tạ` require enough sample size and matching evidence.

## Selection Terms

- `frequency` is the rule's expected natural availability:
  - `always`: almost always has a candidate, for example top ELO or rank leader.
  - `frequent`: appears often with normal data, so it should not dominate the feed.
  - `occasional`: appears sometimes when a real pattern exists.
  - `rare`: unusual event; should be easier to surface when it occurs.
- `appearanceRate` is a multiplier after scoring. It is used to dampen rules
  that are always available. Example: `appearanceRate = 0.35` means the final
  score is only 35% of its raw score, so always-on facts do not crowd out rare
  stories.
- Final score shape in code:
  `selectionScore = (baseWeight + rarityBonus + evidenceStrength + surpriseScore - frequencyPenalty) * appearanceRate`.
- The Analysis UI can pass a page-load seed into insight selection. The seed
  adds bounded deterministic jitter to candidate picking/display order so
  similarly strong stories can reshuffle after F5 without using an auto-rotate
  timer.

Selection rules:

- Return about 6-8 insights.
- A player can appear at most twice.
- A player can appear at most once in the same semantic group.
- A global scenario type appears at most once.

## Future Phase: Expected Probability V2

Do not change expected probability while reviewing the current implemented insight
scenarios. The current expected value is intentionally ELO-only: average ELO of
the two-player team versus average ELO of the opposing two-player team.

After the current implemented scenarios are reviewed and expanded to 4-5
sentence variants, add a separate design/implementation phase for expected
probability v2:

- Keep true win rate as `wins / total`; do not call expected probability
  "win rate" in user-facing text.
- Keep the ELO-only probability as the stable baseline.
- Add small, clamped adjustments for pair chemistry, opponent matchup history,
  and recent form.
- Require sample-size thresholds before pair or opponent adjustments can affect
  the probability, so a 1-2 match sample cannot swing the result.
- Because matches have no draw, if one team has an expected probability of 30%,
  the other team is 70%.
- Re-check every insight that uses expectation gaps after this phase, especially
  upset wins/losses, partner impact, opponent impact, and "above/below ELO
  expectation" wording.

## Technical Data Schema (AI Cheat Sheet)

Below are the TypeScript definitions and mathematical constraints powering the system. Use these schemas to understand all metrics mentioned in the tables below.

```typescript
// 0. RAW BASE DATA: The foundation for all calculations (A match record)
export type AnalysisMatch = {
  id: string;
  date: string;              // ISO Date string (e.g. "2026-05-12T15:00:00Z")
  win_1: string;             // Winner 1 Player ID
  win_2: string | null;      // Winner 2 Player ID (for doubles, null for singles)
  lose_1: string;            // Loser 1 Player ID
  lose_2: string | null;     // Loser 2 Player ID (for doubles, null for singles)
  win_score: number;         // Score of the winning team (e.g. 11, 13)
  lose_score: number;        // Score of the losing team (e.g. 9, 4)
  season: string;            // Season identifier
};

// 1. Core aggregate statistics for an individual player
export type PlayerMetrics = {
  id: string;
  name: string;
  total: number;             // Total matches played
  wins: number;              // Total wins
  losses: number;            // Total losses
  winRate: number;           // Win percentage (%)
  money: number;             // Total fine money accumulated from losses
  rating: number;            // Current ELO rating (starts at 1000)
  
  // Tactical Stats:
  attackScore: number;       // RPG-style Attack rating normalized from points scored
  defenseScore: number;      // RPG-style Defense rating normalized from points conceded
  synergyScore: number;      // Average of partner-specific win rates (RPG Synergy rating)
  formScore: number;         // Form score for latest 5 matches (0 to 100)
  
  // Streak and Form:
  streakCount: number;       // Current length of consecutive win/loss streak
  streakType: 'W' | 'L';     // 'W' for Winning Streak, 'L' for Losing Streak
  recentResults: Array<'W' | 'L'>; // Outcome sequence of latest 8 matches (e.g. ['W', 'L'])
  daysAbsent: number | null; // Consecutive calendar days absent from matches
  
  // Match Outcome Sub-counts:
  closeWins: number;         // Matches WON with a margin <= 2 points
  closeLosses: number;       // Matches LOST with a margin <= 2 points
  deuceMatches: number;      // Matches with high tension (winner's score exceeds 11)
  lowScoreLosses: number;    // Defeats where this player's team scored <= 4 points total
  upsetWins: number;         // Underdog wins (Expected Win Probability < 30% based on ELO)
  upsetLosses: number;       // Favorites falling (Expected Win Probability > 70% based on ELO)
  alternations: number;      // Number of alternating transitions (W to L or L to W) in last 8 games

  // Proposed V4 Expansion Variables (Designed by AI Agent):
  uniqueDays?: number;                 // Total calendar days (sessions) actively played
  matchesPerSession?: number;          // Average matches plowed per session (total / uniqueDays)
  attendanceConsistencyScore?: number; // Schedule consistency based on gap standard deviation (lower = spontaneous)
  Rank?: number;                       // Real Leaderboard Rank (WinRate -> Wins -> Losses)
  eloRank?: number;                    // Hypothetical Rank calculated strictly by ELO score
  previousRank?: number;               // Leaderboard Rank snapshot from previous play day/session
  daysAtTop1?: number;                 // Consecutive days maintaining the #1 Leaderboard Rank
  winRateWithoutPartner?: Record<string, number>; // WinRate (%) when NOT playing paired with a specific partnerID
  tightMatches?: number;               // Total matches (won or lost) with a final score difference <= 3 points
  avgLossDiff?: number;                // Average score difference in losses (e.g., losing 9-11 = diff 2)
  winsVsBottom2?: number;              // Total wins when facing the two lowest-ranked active players (Rank 6 & 7)
  uniquePartners?: number;             // Total unique partners played with (distinct partnerIDs)
  recentLossesVsBottom2?: number;      // Number of defeats against Bottom 2 (Rank 6/7) in the latest 10 matches
  goldenPickled?: number;              // Total historical games lost with 0 points ("Golden Pickle")
};

// 2. Statistics representing relationship between 2 players (Partner or Opponent)
export type AnalysisEdge = {
  kind: 'partner' | 'opponent';
  playerId: string;          // Origin Player ID (Player A)
  playerName: string;
  otherId: string;           // Intersecting Player ID (Player B)
  otherName: string;
  total: number;             // Total shared matches played together (or against each other)
  wins: number;              // Player A's wins in this relationship
  losses: number;            // Player A's losses in this relationship
  rate: number;              // Win Rate of A in this relationship (%)
  impact: number;            // Difference between actual win rate and ELO-expected win rate
  closeGames: number;        // Matches with score margin <= 2 points between them
  deuceGames: number;        // Matches extending past 11 points between them
};
```
